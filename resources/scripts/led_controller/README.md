# LED Controller

Тека містить такі файли:   
* *led_controll.py* — скрит з логікою
* *led_launch.py* — скрипт-керуючий запуском
* *blink.json* — файл-приклад 

Скрипт дозволяє керувати миготінням led-імітаторів. Працює з **.json** файлом в якому прописаний патерн. Патерн представлений у вигляді строки з чотиризначних чисел які візуадльно допомагають швидко зрозуміти, який саме led-маячок увімкнений(**1**), а який вимкнений(**0**).
Поточна модель використовує PYGR-маркер: перший біт керує фіолетовим target-маячком, другий біт керує жовтим distance-маячком, третій біт керує зеленим command-маячком, четвертий біт керує червоним command-маячком.

## Приклад файлу:
```json
{
  "swarm_missions": {
    "my_custom_point": {
      "delay_time": 0.25,
      "loop": true,
      "timeline": [
        {"mask": "1111", "ticks": 4},
        {"mask": "0000", "ticks": 4},
        {"mask": "1100", "ticks": 2}
      ]
    }
  }
}
```
* **"swarm_missions"**  — назва 
* **"my_custom_point"** — фраза з якої може початися відпрацювання скрипта(нижче буде приведено приклад використання).
* **"delay_time": 0.25** — пауза між тіками, в секундах. 0.25 означає 4 рази на секунду.
* **"loop": true** — цикл запускати чи false, не запускати, а пробігтися 1 раз по списку.
* **"mask": "1111"** — FOLLOW: усі чотири PYGR-маячки увімкнені.
* **"mask": "1100"** — SAFE: фіолетовий target і жовтий distance маячки увімкнені, зелений і червоний command-маячки вимкнені.
* **"ticks": 4** — число, скільки триває патерн, час тривалості залежить від значення "delay_time", в даному прикладі, 0.25 х 4, 1 секунда.

## Скрипт запускається наступним чином:
Новий терінал, в якому слід ввести
```sh
# У кожному терміналі в якому
# буде запускатися ROS2 необхідно:
source /opt/ros/humble/setup.sh

# Перехід до теки зі скриптом
cd ~/falcon_gaze/scripts/led_controller

# Далі:
ros2 launch led_launch.py id:=0 file:=blink.json point:=my_custom_point
```
* **led_launch.py** — назва скрипта
* **id:=0** — індексний 
номер дрона, 0 — лідер, наступні 1, 2, 3

#### Схема розташування дронів за індексом в сцені:
``` 
  0
1 2 3
```
* **file:=blink.json** — зміна, назва файлу з патерном. Файл слід зберігати в тій самій теці, що й скрипт. Для відкриття та запуску іншого файлу треба написати назву файлу в змінну без кавичок.

* **point:=my_custom_point** — точка з якої запускається файл. Кожна точка може бути зі своїм набором патернів, чи буде цикл, яка пауза між тіками.
